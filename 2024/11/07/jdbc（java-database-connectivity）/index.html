JDBC（Java Database Connectivity）

Posted on 2024-11-07 Edited on 2024-11-16 In 技术分享

JDBC（Java Database Connectivity） 是 Java 提供的一个用于访问和操作数据库的 API（应用程序编程接口），它使得 Java 程序可以与各种关系型数据库（如 MySQL、PostgreSQL、Oracle、SQL Server 等）进行交互。通过 JDBC，Java 应用程序可以执行 SQL 查询、更新数据库数据、调用存储过程、管理事务等。

JDBC 是 Java 标准库的一部分，它提供了一个通用接口，使得开发者能够以一种标准的方式连接数据库，无需关心底层数据库的具体实现细节。不同的数据库厂商（如 MySQL、Oracle）会为 JDBC 提供各自的驱动程序，Java 程序通过这些驱动程序与数据库进行交互。

JDBC 的核心组成

JDBC 的架构由以下几个核心组件组成：

  1. DriverManager
    DriverManager 类负责管理不同的数据库驱动，Java 程序通过它来注册和连接数据库驱动。DriverManager 会根据提供的数据库 URL 找到合适的驱动来建立连接。

  2. Connection
    Connection 接口表示一个数据库连接对象，提供了与数据库交互的会话。通过 Connection 对象，程序可以发送 SQL 语句、执行事务、设置自动提交等。

  3. Statement
    Statement 是用于执行 SQL 语句的接口，它包括三种常用形式：

    • Statement：用于执行简单的 SQL 语句。
    • PreparedStatement：预编译的 SQL 语句，用于执行带参数的查询，效率更高且防止 SQL 注入。
    • CallableStatement：用于执行数据库存储过程。

  4. ResultSet
    ResultSet 表示查询的结果集，存储着从数据库中查询到的数据。通过 ResultSet 对象，程序可以逐行读取查询结果，访问数据并进行处理。

  5. SQLException
    SQLException 是 JDBC 中用于处理数据库错误的异常类。当数据库访问出错时，JDBC 会抛出此异常，并提供详细的错误信息。

JDBC 的工作流程

  1. 加载数据库驱动
    使用 Class.forName() 方法加载数据库驱动程序，或使用 JDBC 4.0 以后的自动加载功能（无需手动加载）。

  2. 建立数据库连接
    使用 DriverManager.getConnection() 方法，传入数据库 URL、用户名和密码，获取数据库连接对象 Connection。

  3. 创建 SQL 语句对象
    使用 Connection 对象的 createStatement() 方法或 prepareStatement() 方法来创建 SQL 语句对象 Statement 或 PreparedStatement。

  4. 执行 SQL 语句
    使用 Statement 或 PreparedStatement 对象的 executeQuery()（用于查询）或 executeUpdate()（用于插入、更新、删除）方法来执行 SQL 语句。

  5. 处理结果集
    如果执行的是查询语句，则会返回 ResultSet 对象，通过遍历 ResultSet 获取查询到的数据。

  6. 关闭资源
    完成操作后，关闭 ResultSet、Statement、Connection 等资源，避免资源泄漏。

JDBC 的重要特性

  1. 数据库独立性：通过 JDBC API，可以与任何支持 JDBC 的数据库进行交互，只需要切换驱动程序即可。
  2. SQL 注入防护：使用 PreparedStatement 可以防止 SQL 注入攻击，因为它会对参数进行预编译和转义。
  3. 事务管理：通过 Connection 接口，可以启用和控制事务（如提交、回滚），以确保数据一致性。
  4. 批处理：Statement 和 PreparedStatement 支持批量执行多条 SQL 语句，减少数据库交互的次数，提高效率。
  5. 异步处理：JDBC 4.1 之后提供了异步 API，可以异步执行数据库操作，减少阻塞，提高性能。

JDBC 驱动类型

JDBC 驱动程序分为以下四种类型：

  1. Type-1 驱动（JDBC-ODBC Bridge Driver）
    使用 ODBC 驱动，适用于连接不直接支持 JDBC 的数据库，效率较低，已逐渐淘汰。

  2. Type-2 驱动（Native-API Driver）
    使用数据库的本地 API，通过本地库与数据库交互，需要安装数据库的客户端。

  3. Type-3 驱动（Network Protocol Driver）
    使用中间件服务器，通过网络协议连接数据库，适合大型企业环境。

  4. Type-4 驱动（Thin Driver）
    纯 Java 实现的驱动，直接通过数据库协议与数据库通信，是最常用的驱动类型，如 MySQL 和 PostgreSQL 的 JDBC 驱动。

JDBC 优缺点

优点：

  • 提供了标准化的 API，支持多种数据库。
  • 支持事务控制、批处理、存储过程等高级功能。
  • 独立于具体数据库，实现数据库无关性。

缺点：

  • 代码冗长，手动管理连接资源较为复杂。
  • 执行原始 SQL，可能影响代码的可维护性和数据库的可移植性。
  • 数据库操作涉及 I/O，可能会影响性能，通常与连接池和 ORM 框架结合使用。

JDBC 和其他数据访问技术的关系

虽然 JDBC 是基础的数据访问 API，但在实际开发中，很多开发者会选择更高级的 ORM 框架（如 Hibernate、JPA）来简化操作，自动管理对象和表之间的映射关系。然而，JDBC 仍然是 Java 数据库编程的基础，其规范与底层实现为所有高级数据访问技术提供支持。
